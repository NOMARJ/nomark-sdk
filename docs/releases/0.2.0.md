# @nomark-ai/pro 0.2.0 — Intents resolvers

**Released:** 2026-04-18
**Install:** `npm install @nomark-ai/engine @nomark-ai/pro`
**License:** [Business Source License 1.1](../../packages/pro/LICENSE) · source-available, converts to Apache 2.0 on 2029-04-06.

0.2.0 ships the full Intents resolver family — seven target backends that turn a NOMARK Intents `Composition` into runnable source. Six compute emitters (`typescript`, `python`, `rust`, `sql-postgres`, `sql-sqlite`, `sql-mysql`) plus one surface emitter (`react`).

## What's new

### Compute resolvers (`F-RESOLVER`)

`resolve(composition, target)` emits executable source for the six compute backends. Each backend handles the 20 compute verbs (`FETCH`, `MAP`, `FILTER`, `REDUCE`, `VALIDATE`, `PERSIST`, `EMIT`, …) and strips the 11 surface verbs with a `VERBS_STRIPPED` count warning.

```ts
import { resolve } from '@nomark-ai/pro'

const out = resolve(myEtl, 'typescript')
// out.files = [{ path: 'my_etl.ts', content: '...runnable TS...' }]
```

### React surface resolver (`F-SURFACE-RESOLVER`)

`resolve(surfaceComposition, 'react')` emits a `Dashboard.tsx` React component composed from the 5 UI surface verbs (`MONITOR`, `ARRANGE`, `DISPLAY`, `STATUS`, `GUIDE`). Deterministic template emit — single-quoted JSX attributes, 2-space indent, no Prettier pass. The output is byte-stable.

```ts
const ui = resolve(myDashboard, 'react')
// ui.files = [{ path: 'Dashboard.tsx', content: '...JSX component...' }]
```

### `resolveAll` mixed-family manifests

One call across any mix of compute and surface targets, returning a shipping manifest with per-target file paths, byte counts, and warnings.

```ts
import { resolveAll } from '@nomark-ai/pro'

const manifest = resolveAll(myPipeline, ['typescript', 'python', 'react'], {
  basePath: './out',
})
// manifest.targets[i] = { label, files: [{ path, bytes }], warnings: [] }
```

### Dispatcher ambiguity guard

Calling `resolve()` with a `TargetTag` object that sets both `compute` and `surface` now throws rather than silently routing. Pass a label string instead — the error message says so.

### BaseResolver generalisation

Surface backends share the same `BaseResolver` contract as compute backends. One dispatch loop, one warning format, identical ergonomics. External authors can register backends via `registerResolver(label, () => new MyBackend())`.

## Registered targets (7)

| Label | Kind | Output |
|---|---|---|
| `typescript` | compute | `*.ts` with async runtime + topological dispatcher |
| `python` | compute | `*.py` with asyncio runtime, PEP 8 spacing |
| `rust` | compute | `*.rs` + `Cargo.toml`, tokio async runtime |
| `sql-postgres` | compute | `*.sql` with `ON CONFLICT` upsert |
| `sql-sqlite` | compute | `*.sql` with `ON CONFLICT` upsert |
| `sql-mysql` | compute | `*.sql` with `ON DUPLICATE KEY` upsert |
| `react` | surface | `Dashboard.tsx` with 5 surface-verb handlers |

- Requires `@nomark-ai/engine >= 0.1.0` (unchanged).
- Node.js ≥ 18.
